--- layout: single author_profile: false title: "SENSE" --- # SENSE: 재난 대응 지능형 에이전트 오케스트레이션 - 작성일: {{ site.time | date: '%Y-%m-%d' }} - 버전: 1.0 ## 목차 - [프로젝트 미션과 문제의식](#프로젝트-미션과-문제의식) - [활용 시나리오](#활용-시나리오) - [팀의 기술적 강점](#팀의-기술적-강점) - [기대 효과와 임팩트](#기대-효과와-임팩트) - [전체 아키텍처 흐름도](#전체-아키텍처-흐름도) - [부록: 세부 아키텍처와 데이터 플로우](#부록-세부-아키텍처와-데이터-플로우) ## 프로젝트 미션과 문제의식 재난 현장은 "정보의 단절"과 "의사결정 지연"이 큰 피해로 직결됩니다. 센서, CCTV, 공공데이터, 시민제보 등 다양한 데이터가 흩어져 있고, 대응 조직 간(소방/경찰/지자체/병원) 조율이 어려워 실시간 최적 행동 도출이 지연됩니다. SENSE의 미션은 다음과 같습니다: - 분산된 데이터를 표준화·통합하고, 멀티 에이전트가 실시간으로 상황을 이해·추론·행동 - 현장과 관제, 자동화와 인간 의사결정을 유기적으로 결합하는 오케스트레이션 - 데이터/모델/시스템의 이기종성을 흡수하는 모듈식 아키텍처로 신속한 확장·배포 지원 왜 필요한가? - 분 단위가 생명을 좌우: 탐지→판단→행동의 지연을 초 단위로 단축 - 불확실성 고리 제거: 다원 데이터 융합과 추론형 에이전트로 오탐/미탐 감소 - 협업의 비용 절감: 조직 간 인터페이스 표준화와 역할 기반 워크플로우 자동화 ## 활용 시나리오 1) 지진 발생 시 - 센싱: 가속도계/USGS 피드/CCTV 흔들림 패턴을 스트리밍 수집 - 분석: 다중 센서 융합으로 진도·피해 추정, 건물 붕괴 위험도 예측(구역 그리드 기반) - 행동: 대피 경로 제안(도로 파손/정체 반영), 병상 가용 병원 자동 매칭, 드론 정찰 투입 계획 - 조율: 소방·경찰·지자체 간 우선순위 티켓 발행 및 중복 출동 제거 2) 도심 화재 - 센싱: 화염/연기 비전모델, 화재감지기, 시민 112 사진 제보 자동 수집 - 분석: 바람/내부 평면도/가연물 정보로 연소 확산 시뮬레이션, 인명 밀집도 추정 - 행동: 층별 대피 방송 스크립트 생성, 스프링클러 제어 API 호출, 교통 신호 최적화로 소방차 골든타임 확보 - 조율: 대피 유도 요원·소방·병원 이송 라우팅을 하나의 플레이북으로 동기화 3) 국지성 폭우/홍수 - 센싱: 강우 레이더, 하천 수위, 하수관 IoT, 침수 신고 콜 데이터 스트림 - 분석: 격자 기반 침수 예측(15~30분 선행), 위험 구역 알림의 정밀 지오펜싱 - 행동: 우선 배수펌프 가동·차단기 제어, 우회 도로 네비게이션 링크 푸시, 취약계층 안부콜 자동화 - 조율: 구청·도로공사·교통경찰 작업 큐를 통합 스케줄링 ## 팀의 기술적 강점 - 에이전트 오케스트레이션: LangGraph/AutoGen 기반 다중 에이전트(감지·분석·행동·조율) with 역할·권한(RBAC)과 토큰예산 관리 - 실시간 데이터 파이프라인: Kafka/RabbitMQ 기반 스트리밍, Flink/Spark 스트리밍으로 윈도우 집계·이상탐지 - 지리공간 추론: PostGIS/pgRouting, H3 격자, ETA 기반 경로 탐색 및 위험도 히트맵 - 시뮬레이션·예측: 시계열 예측(Prophet/Temporal Fusion Transformer), 화염 확산 CA/CFD 경량 모델 - 견고한 운영: Kubernetes, GitHub Actions(CI/CD), Prometheus+Grafana, 카나리 릴리즈와 롤백 - 차별점 - 규칙 기반+LLM 하이브리드로 "설명 가능한 행동 계획" 생성 - 인간-루프(HITL) 승인 게이트 내장으로 오탐 위험 제어 - 기존 관제는 단일 벤더·폐쇄형: SENSE는 오픈 인터페이스, 모듈식, 멀티벤더 친화 ## 기대 효과와 임팩트 - 대응 시간 단축: 감지→조치까지 평균 40~70% TTR 감소 목표 - 인명 피해 최소화: 정밀 대피·자원배분으로 사상자율 감소 - 예산 효율화: 중복 출동·오경보 비용 절감, 표준 API로 벤더 종속 축소 - 데이터 거버넌스: 감사 로그·정책 기반 접근 제어로 공공 데이터 안전 활용 - 사회적 신뢰: 투명한 근거 제시와 사후 재현 가능한 리포트 자동 생성 ## 전체 아키텍처 흐름도 아래 이미지는 SENSE의 전체 오케스트레이션 흐름을 보여줍니다. assets/images/ 디렉터리에 flow-architecture.png 파일을 업로드하고 경로를 확인하세요. ![SENSE 전체 흐름도](assets/images/flow-architecture.png) > 이미지가 보이지 않으면: 저장소에 assets/images/flow-architecture.png를 업로드한 뒤 다시 빌드하세요. ## 부록: 세부 아키텍처와 데이터 플로우 ### 레이어 구성 -
